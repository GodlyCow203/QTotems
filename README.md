# QTotems

Small Minecraft plugin that provides configurable "totems" which grant effects when equipped or popped.

This README documents features, commands, permissions, configuration and troubleshooting steps.

---

## Features

- Config-driven totems under `totems:` in `config.yml`.
- Each totem can provide:
  - Equip effects (applied while the totem is equipped)
  - Pop effects (applied when the totem is consumed/popped)
  - Custom lore and display name
  - Ability to enable/disable individual totems via `enabled: false`
- Central `prefix` and configurable command/messages under `messages:` for easy localization.
- Safe handling when `totems:` is missing (plugin logs a warning instead of throwing an exception).
- Console-support for the `reload` subcommand.

---

## Commands

- `/totems` (main command)
  - Aliases: `/qtotems`, `/qtotem` (declared in `plugin.yml`)
  - Usage:
    - `/totems` — shows usage message
    - `/totems reload` — reloads plugin config and totems (can be run from console)
    - `/totems <totem>` — gives you the named totem item
    - `/totems <totem> <player>` — gives the named totem to `<player>`

Examples:
```
/totems totem1
/totems totem1 SomePlayer
/totems reload
```

## Permissions

- `qtotems.command` — required to use the `/totems` command.
  - Default: `op` (see `src/main/resources/plugin.yml`)

You can change permission requirements by editing `plugin.yml` or integrating with your permissions plugin.

## Configuration (`config.yml`)

The plugin ships with a simple `config.yml` under `src/main/resources/config.yml`. Key sections:

- `prefix` — text prefixed to messages (MiniMessage/legacy color codes supported)
- `totems:` — map of totem ids to their configuration
  - `<id>:`
    - `enabled: true` — set to `false` to skip registration of that totem
    - `name:` display name (optional; the key is used if missing)
    - `lore:` list of lore lines
    - `popEffects:` list of strings in the format `EFFECT;amplifier;duration` (duration in ticks or seconds depending on implementation)
    - `equipEffects:` list of strings in the format `EFFECT;amplifier`

- `messages:` — all messages shown by the command
  - `onlyPlayers`, `usage`, `reloaded`, `invalidTotem`, `gaveSelf`, `gaveTarget`, `invalidTarget` are provided by default

Example snippet:
```yaml
prefix: "<dark_red>[<red><bold>QTotems</bold></red>]</dark_red> "

totems:
  totem1:
    enabled: true
    name: "Totem of Example"
    lore:
      - "A simple example totem"
    popEffects:
      - "SLOW_FALLING;1;60"
    equipEffects:
      - "STRENGTH;1"

messages:
  onlyPlayers: "<red>Only players can use this command!"
  usage: "<yellow>Usage: /qtotems <totem> {player}"
  reloaded: "<green>Reloaded config!"
```

Notes on effects:
- `popEffects` entries are expected to be semicolon-separated values. Invalid lines are ignored and logged.

## Behavior & Safety

- If `totems:` is missing from the config, the plugin logs a clear warning and registers no totems (no NullPointerException).
- Each totem supports an `enabled` flag; disabled totems are skipped during registration and a message is logged.
- `Utils.textWithPrefix(...)` safely falls back when `prefix` is missing to avoid seeing `null` in messages.

## Installation & Development

Build the plugin with Gradle and copy the produced jar to your server's `plugins/` folder:

```fish
./gradlew build --no-daemon
# jar will be in build/libs/
```

Restart or reload the server. On first run the bundled `config.yml` is copied to `plugins/QTotems/config.yml`.

---

## Troubleshooting

- "No 'totems' section found in config.yml" — your `config.yml` is missing the `totems:` section. Restore the default `config.yml` or add a `totems:` section.
- "Skipping disabled totem: <id>" — that totem has `enabled: false`.
- If messages include `null`, ensure `prefix` is present in `config.yml` (the plugin now falls back to an empty prefix, but older logs/messages may show it).
